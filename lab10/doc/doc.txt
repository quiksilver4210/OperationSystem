Задание 4:
Планирование по принципу FIFO (first-in-first-out)

Принцип FIFO, «первый пришедший обслуживается первым», является наиболее простой
дисциплиной планирования. ЦП предоставляется процессам в порядке их прихода в очередь готовности.
После того, как процесс получает ЦП в свое распоряжение, он выполняется до завершения

Время ожидания - не оптимальное.
Повышение приоритета "забытой" задачи - невозможно.
Время завершения - зависит от разположения процессов.
Вариативность времени завершения - высокая.


Циклическое планирование round robin (RR)

Планирование по принципу RR предполагает диспетчирование процессов по принципу FIFO,
но каждый процесс получает временной квант, в течение которого он может использовать ресурсы ЦП.
Если завершения процесса не происходит по истечении кванта времени, то этот процесс переводится
в конец списка готовых к выполнению процессов, а ресурсы ЦП предоставляются следующему процессу из списка.
Такой алгоритм планирования подходит, например, для работы с разделением времени,
когда система должна гарантировать приемлемые времена ответа для всех интерактивных пользователей.

Время ожидания - не оптимальное.
Повышение приоритета "забытой" задачи - невозможно.
Время завершения - большое.
Вариативность времени завершения - зависит от кванта.

Shortest Job First (SJF)

Обслуживание самого короткого задания первым. Стратегия диспетчеризации процессора, при которой
процессор предоставляется в первую очередь наиболее короткому процессу из имеющихся в системе.

Время ожидания(среднее) - оптимально.
Повышение приоритета "забытой" задачи - возможно.
Время завершения - оптимально.
Вариативность времени завершения - невозможна для данного набора процессов.
